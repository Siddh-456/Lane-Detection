Theory Reference
What is Lane Detection Studio?
A production-ready deep learning system for road lane detection and segmentation.

This project implements LaneNet, based on the IEEE IV 2018 paper "Towards End-to-End Lane Detection: an Instance Segmentation Approach". It uses a lightweight ENet encoder combined with dual decoders to produce both binary segmentation masks and coloured instance segmentation outputs simultaneously from a single forward pass.

The system is trained on the TuSimple benchmark and served via a Flask REST API. A threading lock prevents concurrent GPU allocation. Pre-trained weights are loaded from log/best_model.pth.

Binary Segmentation
Classifies every pixel as lane or non-lane. Uses Focal Loss to handle severe class imbalance — lanes occupy only a small fraction of any road image.
Instance Segmentation
Assigns a unique identity to each individual lane using Discriminative Loss embeddings. No explicit instance labels required during training.
Input / Output
Input: RGB images resized to 512 × 256. Output: 2-channel binary mask + 3-channel RGB instance colour map.
512 × 256 px
Deployment
Flask web server with Docker support. REST endpoint at /predict. Supports local, AWS EC2, Heroku, and Kubernetes deployments.

Model Architecture
A shared encoder feeding two independent decoder branches.

LaneNet uses a single shared encoder that feeds into two independent decoders — one for binary segmentation (lane vs background) and one for instance embeddings. The shared encoder allows the model to learn common low-level features efficiently, while each decoder specialises for its task.

ENet — Default
~3M parameters. Uses an InitialBlock with parallel conv + maxpool, followed by BottleneckModules with dilated and asymmetric convolutions. PReLU activations, BatchNorm2d, and dropout for regularisation. Designed for real-time inference on edge hardware.
~3M params · 25ms · Default
U-Net — Balanced
~30M parameters. 5-level encoder with DoubleConv blocks (two paired Conv2d + BN + ReLU). Progressive channel expansion: 64→128→256→512→1024. Skip connections from encoder to decoder preserve fine spatial detail lost during downsampling.
~30M params · 45ms
DeepLabv3+ — Premium
~50M parameters. ResNet-101 backbone with Atrous Spatial Pyramid Pooling (ASPP) using dilation rates 6, 12, 18 across five parallel branches plus global average pooling. Shortcut low-level features (48 channels) from the encoder. SynchronizedBatchNorm for multi-GPU training.
~50M params · 80ms · Highest Accuracy

Normalisation stats for all backbones: mean [0.485, 0.456, 0.406], std [0.229, 0.224, 0.225] — standard ImageNet statistics for compatibility with pre-trained encoder weights.

Training the Model
Loss functions, dataset preparation, and optimisation strategy.

The model is trained end-to-end with a combined loss that weights binary detection 10× more than instance separation, reflecting the primacy of detecting lane pixels before differentiating them from one another.

Focal Loss — Binary Segmentation
FL(pₜ) = −α(1−pₜ)^γ · log(pₜ)    γ=2, α=[0.25, 0.75]
Down-weights the contribution of easy, correctly-classified background pixels so the model focuses its learning capacity on the rare and difficult lane pixels. Weight in combined loss: 10×.
Discriminative Loss — Instance Segmentation
L = L_var + L_dist + L_reg    δ_var=0.5, δ_dist=1.5, γ=0.001
L_var pulls pixels of the same lane close to their mean embedding. L_dist pushes different lane mean embeddings apart. L_reg prevents embeddings growing unbounded. Combined weight: 0.3 × L_var + 1.0 × L_dist.

Total Loss = 10 × FocalLoss + 0.3 × L_var + 1.0 × L_dist

Dataset — TuSimple Benchmark. Each training sample contains an RGB image, a binary ground-truth mask (lane pixel = 1, background = 0), and an instance mask where each lane is assigned a unique colour. Data is generated with tusimple_transform.py which supports training-only, train+val, or train+val+test splits.

Data Augmentation (training only): Resize to 256×512 → ColorJitter (brightness, contrast, saturation, hue each ±0.1) → ToTensor → ImageNet Normalize. Validation uses resize and normalise only — no augmentation. Label masks are resized with INTER_NEAREST to preserve discrete integer values.

Optimiser: Adam or SGD with step/exponential learning rate scheduling. Default: 100 epochs, batch size 32, lr = 0.001, weight decay = 1e-4, validation split = 10%.

Inference Pipeline
From browser upload to JSON response — step by step.
1
Image Upload
User submits a multipart form to POST /predict. Flask reads the file from request.files["image"].
2
Acquire Inference Lock
INFERENCE_LOCK (threading.Lock) prevents simultaneous forward passes, avoiding GPU memory conflicts on shared hardware.
3
Preprocess
PIL opens the image. Resize to 512×256 → ToTensor → ImageNet Normalize. Tensor moved to CUDA if available, else CPU.
4
Forward Pass
LaneNet forward pass produces binary logits (2 channels) and instance embeddings (3 channels). Weights from log/best_model.pth.
5
Post-process
Argmax over binary logits produces the lane mask. Instance embeddings are colour-mapped to a 3-channel RGB output. Results saved to test_output/.
6
Encode & Respond
Both images are base64-encoded and returned as: { "binary_image": "...", "instance_image": "..." }. The frontend decodes and displays them inline.

Evaluation Metrics used throughout training:

Dice Coefficient (F1-score)
Dice = 2 · |A∩B| / (|A| + |B|)
Range [0,1]. More robust to class imbalance than pixel accuracy. Threshold applied at 0.5 for binarising predictions. Preferred metric for lane evaluation.
Intersection over Union (IoU)
IoU = |A∩B| / |A∪B|
Range [0,1]. Stricter than Dice — penalises false positives and false negatives equally. Standard segmentation benchmark metric.
