What is Lane Detection Studio?

A compact inference interface for exploring LaneNet-style road lane segmentation in real time.

This project packages LaneNet into a clean browser-based workflow. A single forward pass generates both a binary segmentation mask and an instance-aware lane embedding map, making it easy to inspect where the network sees drivable lane markings and how it separates one lane from another.

The current stack is tuned for the TuSimple benchmark, served through a Flask API, and backed by weights loaded from log/best_model.pth. The frontend stays lightweight while the backend handles image normalization, model execution, and base64 output encoding.

Binary Segmentation

Every pixel is classified as lane or background. This branch focuses on recall and sharp lane boundaries, even when the foreground occupies only a very small area of the frame.

Lane / Background

Instance Segmentation

Embedding vectors allow the system to distinguish adjacent lanes without requiring a separate object detector. Clustering in embedding space produces visually distinct lane groups.

Lane Identity

Input Resolution

Inference runs on frames resized to 512 × 256, balancing edge-device throughput and enough spatial fidelity for lane structure.

Model Architecture

A shared encoder, dual decoder design optimised for efficient lane understanding.

LaneNet uses a single shared feature extractor so both tasks benefit from the same low-level road texture, edge, and perspective cues. After encoding, the network splits into two task-specific heads: one branch predicts binary logits, and the other predicts dense embeddings used for lane separation.

ENet Backbone

The default model uses ENet for its small footprint and fast inference characteristics. It relies on early downsampling, asymmetric convolution blocks, dilated kernels, and lightweight bottlenecks to keep latency low on constrained hardware.

~3M params · Real-time

Shared Features

Because the binary and embedding heads see the same encoder output, the model can learn lane-relevant structure once and reuse it for both detection and separation. This reduces total compute compared with training two independent networks.

Shared Encoder

Normalisation

Image preprocessing uses ImageNet mean and standard deviation values: mean [0.485, 0.456, 0.406] and std [0.229, 0.224, 0.225]. That keeps the input scale compatible with pre-trained encoder expectations.

Training Strategy

The model is trained end-to-end with a weighted combination of detection and embedding losses.

Lane pixels are sparse compared with road background, so the binary head uses Focal Loss to keep the network focused on difficult, underrepresented foreground examples. The instance branch uses a Discriminative Loss that pulls same-lane embeddings together while pushing different-lane clusters apart.

Focal Loss
FL(p_t) = - alpha (1 - p_t)^gamma log(p_t), gamma = 2
This loss down-weights easy background predictions and makes mislabeled or low-confidence lane pixels matter more during optimization. In the combined objective, the binary term carries the highest weight.
Discriminative Loss
L = L_var + L_dist + L_reg, delta_var = 0.5, delta_dist = 1.5
L_var compresses embeddings within a lane cluster, L_dist separates different lane centers, and L_reg prevents unbounded feature drift. Together they produce stable lane instances without a classic object detector.

Typical training setup: 100 epochs, batch size 32, learning rate 0.001, and augmentation with resize plus lightweight color jitter. Labels are resized with nearest-neighbor interpolation so discrete class IDs remain intact.

Inference Pipeline

From browser upload to rendered output in six steps.

1
Frame Upload

The browser sends a multipart form request to /predict with the selected road image under the image field.

2
Input Sanitisation

Flask loads the upload through Pillow, normalises orientation metadata, and converts the image into a clean three-channel RGB JPEG for the existing inference path.

3
Locked Execution

A global threading.Lock prevents overlapping model runs, which helps avoid memory contention on GPU-backed deployments.

4
Forward Pass

LaneNet generates binary logits and instance embeddings from the resized input tensor, using weights stored at log/best_model.pth.

5
Post-processing

The binary mask is produced from class predictions while the embedding output is converted into a colored visualization so each lane group is easier to inspect.

6
JSON Response

The backend returns base64-encoded binary_image and instance_image fields, and the frontend injects them directly into the preview panels.

Dice Coefficient
Dice = 2 |A intersect B| / (|A| + |B|)
Dice is commonly preferred for lane evaluation because it remains informative even when the positive class occupies only a small fraction of the frame.
Intersection over Union
IoU = |A intersect B| / |A union B|
IoU is stricter than Dice and penalizes false positives and false negatives symmetrically, making it a strong benchmark metric for segmentation quality.
